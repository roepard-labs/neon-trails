Arquitectura del sistema

Patrón cliente-servidor con API REST JSON: el frontend Vue consume la API Laravel; la capa de broadcasting notifica cambios de tareas por WebSockets (Reverb) a clientes suscritos vía Echo.

Backend (Laravel)

  • Rutas /api/* con middleware auth:sanctum en recursos protegidos.
  • Autenticación por token; verificación de correo en flujos registrados.
  • Scramble (dedoc/scramble): UI en /docs/api, especificación OpenAPI en /docs/api.json; atributos @Endpoint en controladores; acceso con SCRAMBLE_PUBLIC_DOCS / SCRAMBLE_ALLOWED_EMAILS en config/scramble.php.
  • Eventos de dominio hacia canales tasks y user.{id}.tasks.

Frontend (Vue 3)

  • Pinia para estado (auth, tasks); Axios con token en Authorization.
  • Vue Router (src/router/index.js): historial HTML5, rutas nombradas y meta.requiresAuth / meta.guestOnly; guarda global beforeEach.
  • Echo escucha TaskCreated, TaskUpdated, TaskDeleted.
  • UI con componentes tipo “card” y hero alineados a la identidad del producto.

Documentación OpenAPI (Scramble)

Scramble analiza rutas y controladores y expone una UI interactiva más el JSON OpenAPI 3.

  • GET /docs/api — interfaz de documentación (Scramble UI)
  • GET /docs/api.json — especificación OpenAPI generada
  • GET /docs → redirección a /docs/api (routes/web.php)
  • /api/documentation → /docs/api · /api/documentation.json → spec JSON (routes/api.php, prefijo /api)
  • Export estático: php artisan scramble:export --path=storage/api-docs/api-docs.json
  • Análisis CI: php artisan scramble:analyze

Rutas Vue (Vue Router)

Fuente: task-manager-web/src/router/index.js. Base: createWebHistory(import.meta.env.BASE_URL).

Ruta Nombre Meta / notas
/ guest.home Landing; enlace a docs abre VITE_API_URL + '/docs'
/tasks tasks.index requiresAuth: true — sin token → redirección a guest.home
/verify-email auth.verify-email Flujo de verificación de correo (query status)

beforeEach: si la ruta tiene requiresAuth y no hay token en localStorage, navega a guest.home. Si guestOnly y hay sesión: si el email no está verificado → auth.verify-email; si está verificado → tasks.index.

Endpoints API relevantes

  • GET /api/status — estado del servicio
  • POST /api/register · POST /api/login · POST /api/logout
  • GET|POST /api/tasks · GET|PUT|PATCH|DELETE /api/tasks/{id}
  • POST /api/broadcasting/auth — autorización de canales privados (Sanctum)
  • GET /api/email/verify/{id}/{hash} — verificación firmada
