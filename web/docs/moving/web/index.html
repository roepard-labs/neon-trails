Proyecto integrador

TaskManager

Sistema de gestión de tareas API-first: organizar el trabajo personal o de equipo, con sincronización en tiempo real y enfoque en cerrar tareas pendientes.

Contexto y problema

En entornos académicos y profesionales se pierde visibilidad sobre prioridades, fechas límite y cambios hechos por otros. Se necesita un sistema coherente que separe claramente presentación, lógica de negocio y datos, y que permita evolucionar el frontend sin acoplarlo al detalle del servidor.

  • Objetivo: centralizar tareas con estados, vencimiento y actualizaciones en vivo.
  • Alcance: autenticación con token, CRUD de tareas, canales realtime (Echo + Reverb).
  • Valor: arquitectura desacoplada, documentación OpenAPI y despliegue reproducible.

Requerimientos (marco del curso)

Alineado con la guía académica Requerimientos y casos de uso: los requerimientos se clasifican por nivel (negocio, usuario, sistema) y por tipo funcional (RF) vs no funcional (RNF). Los casos de uso modelan los RF y se documentan aparte (diagrama + texto).

Nivel Pregunta guía Ejemplo en TaskManager
Negocio ¿Por qué existe el sistema? Reducir retrabajo por tareas desactualizadas; visibilidad de vencimientos y estado.
Usuario ¿Qué debe poder hacer el actor? Registrarse, iniciar sesión, CRUD de tareas, ver cambios en tiempo real, consultar API docs.
Sistema ¿Cómo lo cumple técnicamente? REST con Sanctum, Vue Router + Pinia, Reverb/Echo, OpenAPI con Scramble, HTTPS en producción.

RF y RNF aplicados

  • RF: autenticación con usuario/contraseña y token; CRUD de tareas; verificación de correo; broadcasting auth; documentación de endpoints (Scramble).
  • RNF: seguridad (HTTPS, token Bearer, acceso restringido a docs según env); rendimiento razonable vía API paginada; disponibilidad dependiente del despliegue; usabilidad responsive básica en la SPA.

Stack

Laravel 12 (API), Vue 3 + Vite, Sanctum, Laravel Echo + Reverb.

Contrato y docs

La API se describe con OpenAPI generada por Dedoc Scramble (/docs/api, JSON en /docs/api.json). El front abre ${VITE_API_URL}/docs desde HomeView.vue; en Laravel, routes/web.php redirige /docs → /docs/api (UI Scramble). Export CI: php artisan scramble:export --path=storage/api-docs/api-docs.json.

Vue Router en la SPA

Rutas definidas en src/router/index.js con createWebHistory(import.meta.env.BASE_URL): guest.home (/), tasks.index (/tasks, requiresAuth), auth.verify-email (/verify-email). El beforeEach redirige sin token desde rutas protegidas al home y gestiona verificación de correo.
